# Установка AWG Manager на Keenetic

Пошаговая инструкция по установке и настройке AmneziaWG туннелей на роутерах Keenetic.

## Требования

- Роутер Keenetic с установленным Entware
- USB-накопитель, отформатированный в ext4
- Доступ к командной строке роутера (SSH)
- Конфигурационный файл `.conf` от вашего VPN-провайдера (WireGuard или AmneziaWG)

---

## Шаг 1. Установка Entware (если ещё не установлен)

### 1.1. Подготовка USB-накопителя

1. Вставьте USB-накопитель в роутер
2. Откройте веб-интерфейс роутера: `http://192.168.1.1` (или ваш адрес)
3. Перейдите в **Управление** → **Приложения**
4. Нажмите на USB-накопитель и выберите **Форматировать в ext4**

### 1.2. Установка OPKG (Entware)

1. В разделе **Приложения** найдите **Менеджер пакетов OPKG**
2. Нажмите **Установить**
3. Дождитесь завершения установки

---

## Шаг 2. Подключение по SSH

### Windows (с помощью PuTTY)

1. Скачайте [PuTTY](https://www.putty.org/)
2. Запустите PuTTY
3. В поле **Host Name** введите IP роутера: `192.168.1.1`
4. Порт: `22` (или `222` для Entware)
5. Нажмите **Open**
6. Введите логин: `root`
7. Введите пароль (тот же, что для веб-интерфейса)

### macOS / Linux

Откройте терминал и выполните:
```bash
ssh root@192.168.1.1 -p 222
```

---

## Шаг 3. Установка AWG Manager

### Способ 1: Через репозиторий (рекомендуется)

#### 3.1. Добавление репозитория

**Для Keenetic с процессором MIPS (большинство моделей):**
```bash
echo "src/gz awg https://hoaxisr.github.io/entware-repo/mipsel-3.4-kn" >> /opt/etc/opkg.conf
```

**Для Keenetic с процессором ARM (Ultra, Giga, Viva и др.):**
```bash
echo "src/gz awg https://hoaxisr.github.io/entware-repo/aarch64-3.10-kn" >> /opt/etc/opkg.conf
```

> **Как узнать архитектуру?** Выполните `uname -m`:
> - `mips` → используйте mipsel-3.4-kn
> - `aarch64` → используйте aarch64-3.10-kn

#### 3.2. Установка

```bash
opkg update
opkg install awg-manager
```

#### 3.3. Запуск сервиса

```bash
/opt/etc/init.d/S98awg-manager start
```

---

### Способ 2: Ручная установка

Если репозиторий недоступен, скачайте пакет вручную (замените `X.X.X` на актуальную версию):

**Для MIPS:**
```bash
cd /opt/tmp
wget https://github.com/hoaxisr/awg-manager/releases/download/vX.X.X/awg-manager_X.X.X_mipsel-3.4-kn.ipk
opkg install /opt/tmp/awg-manager_*.ipk
```

**Для ARM:**
```bash
cd /opt/tmp
wget https://github.com/hoaxisr/awg-manager/releases/download/vX.X.X/awg-manager_X.X.X_aarch64-3.10-kn.ipk
opkg install /opt/tmp/awg-manager_*.ipk
```

Затем запустите сервис:
```bash
/opt/etc/init.d/S98awg-manager start
```

---

## Шаг 4. Открытие веб-интерфейса

1. Откройте браузер
2. Перейдите по адресу: `http://192.168.1.1:8080`

   > Если порт 8080 занят, попробуйте `http://192.168.1.1:8081`

3. Вы увидите главную страницу AWG Manager

---

## Шаг 5. Импорт конфигурации туннеля

### 5.1. Получение конфигурации

Получите файл `.conf` от вашего VPN-провайдера. Он выглядит примерно так:

```ini
[Interface]
PrivateKey = ваш_приватный_ключ
Address = 10.0.0.2/32
DNS = 1.1.1.1

[Peer]
PublicKey = публичный_ключ_сервера
Endpoint = vpn.example.com:51820
AllowedIPs = 0.0.0.0/0
PersistentKeepalive = 25
```

### 5.2. Импорт в AWG Manager

1. На главной странице нажмите **Создать**
2. Убедитесь, что выбрана вкладка **Импорт .conf**
3. Введите название туннеля (например: "Мой VPN")
4. **Способ 1:** Перетащите файл `.conf` в область загрузки
5. **Способ 2:** Скопируйте содержимое файла и вставьте в текстовое поле
6. Нажмите **Импортировать**

### 5.3. Проверка и сохранение

После импорта откроется страница редактирования:

1. Проверьте, что все поля заполнены корректно
2. При необходимости измените настройки
3. Нажмите **Сохранить**

---

## Шаг 6. Запуск туннеля

### 6.1. Запуск вручную

1. На главной странице найдите ваш туннель
2. Нажмите кнопку **Запустить** (зелёная кнопка с треугольником)
3. Дождитесь появления статуса **Работает**

### 6.2. Включение автозапуска

Чтобы туннель запускался автоматически при включении роутера:

1. На карточке туннеля найдите переключатель справа
2. Включите его (он станет зелёным)
3. Появится уведомление "Автозапуск включён"

---

## Шаг 7. Тестирование соединения

После запуска туннеля проверьте его работу:

1. На карточке туннеля нажмите **Тест**
2. Или перейдите в редактирование и нажмите **Открыть тесты**

### Доступные тесты:

- **Проверка соединения** — проверяет доступ в интернет через туннель
- **Проверка IP** — показывает ваш реальный IP и IP через VPN
- **Тест скорости** — измеряет скорость загрузки через туннель

---

## Шаг 8. Настройка маршрутизации (опционально)

По умолчанию туннель направляет весь трафик через VPN. Если вам нужно направлять только определённые устройства или сайты:

### Через веб-интерфейс Keenetic:

1. Откройте `http://192.168.1.1`
2. Перейдите в **Интернет** → **Другие подключения**
3. Найдите интерфейс `awg0` (или `awg-имя_туннеля`)
4. Настройте правила маршрутизации

---

## Решение проблем

### Туннель не запускается

1. Проверьте, что конфигурация корректна
2. Убедитесь, что endpoint сервера доступен
3. Проверьте логи:
   ```bash
   logread | grep awg
   ```

### Нет доступа к веб-интерфейсу

1. Проверьте, запущен ли сервис:
   ```bash
   /opt/etc/init.d/S98awg-manager status
   ```
2. Перезапустите сервис:
   ```bash
   /opt/etc/init.d/S98awg-manager restart
   ```

### Медленная скорость

1. Попробуйте другой сервер VPN-провайдера
2. Уменьшите MTU до 1280 или ниже
3. Для AmneziaWG: попробуйте разные настройки Jc, Jmin, Jmax

### IP не меняется

1. Проверьте поле **AllowedIPs** — должно быть `0.0.0.0/0, ::/0`
2. Убедитесь, что туннель действительно запущен
3. Проверьте маршрутизацию в настройках роутера

## Обновление

Если вы установили через репозиторий:

```bash
opkg update
opkg upgrade awg-manager
```

Сервис перезапустится автоматически.

---

## Удаление

Для полного удаления AWG Manager:

```bash
# Остановить сервис
/opt/etc/init.d/S98awg-manager stop

# Удалить пакет
opkg remove awg-manager

# Удалить конфигурации (опционально)
rm -rf /opt/etc/awg-manager
```

---

## Полезные команды

```bash
# Статус сервиса
/opt/etc/init.d/S98awg-manager status

# Перезапуск сервиса
/opt/etc/init.d/S98awg-manager restart

# Просмотр логов
logread | grep awg

# Список туннелей
ls /opt/etc/awg-manager/tunnels/

# Статус интерфейса
awg show
```
